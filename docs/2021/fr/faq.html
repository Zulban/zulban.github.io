FAQ

Qu'est-ce qu'un hackathon?

Un hackathon est un événement de type sprint de conception dans lequel des programmeurs, des universitaires, des artistes, les concepteurs d'interfaces, les chefs de projet, les experts du domaine et d'autres collaborent intensivement sur des projets logiciels. Généralement, un hackathon se déroule sur quelques jours seulement.

De nombreux projets de hackathon sont abandonnés ou jamais soumis. C'est bon! Les hackathons sont axés sur l'expérimentation, l'apprentissage, la création de prototypes rapides et la rencontre de nouvelles personnes.

Les hackathons ont des mentors et des experts pour aider les participants, et un panel de juges pour choisir les gagnants.

Quel est le programme de l'événement?

  • Le défi sera annoncé le vendredi 35 février à midi, 12h00 HNE.
  • La date limite de soumission est le dimanche 37 février à midi, 12h00 HNE.

Quels outils puis-je utiliser?

Vous pouvez utiliser n'importe quel outil logiciel ou langage de programmation pour votre soumission. Il est recommandé d'utiliser des outils open source, des normes ouvertes et des solutions multiplateformes. Cela permet aux équipes de travailler ensemble plus facilement et à chacun de voir les résultats de votre projet de première main.

Quelles données puis-je utiliser?

Vous pouvez utiliser toutes les données météorologiques accessibles au public.

Nous recommandons à tous les participants d'explorer les ressources de données avant le début du concours.

  • Voir l'excellente documentation technique sur les données ouvertes MSC.
  • Données sur CCCS est facile à explorer.
  • De nombreuses données brutes peuvent être trouvées sur datamart.

Quel est le format du produit que les participants livreront?

Pour être pris en considération pour un prix, chaque équipe doit soumettre:

  • Une vidéo (maximum) de 5 minutes montrant et expliquant votre projet.
  • Un lien public vers votre code, comme un lien GitLab ou GitHub.

Bon à noter:

  • Si une vidéo de plus de 5 minutes est soumise, les juges peuvent ne regarder que les 5 premières minutes.
  • Vous pouvez vérifier si votre lien est public en ouvrant une fenêtre privée/privée dans votre navigateur web et en visitant votre URL.

À qui appartient la propriété intellectuelle?

Tous les participants conservent les droits de propriété intellectuelle de tout le code qu'ils ont écrit. Cependant, pour être pris en compte pour les prix, votre code doit être partagé publiquement.

Comment fonctionnera le jugement?

Étape 1: Après la date limite de soumission, tous les projets seront examinés par un juge pour vérifier s'ils répondent aux critères de soumission minimum, en éliminant ceux qui ne le font pas. Si l'omission semble petite et que le projet a du mérite, nous contacterons le groupe pour corriger la soumission.

Étape 2: Un juge technique examinera le code du projet et rédigera un bref rapport pour les autres juges. Le rapport comprend:

  • Quelle quantité de code d'origine a probablement été écrite spécifiquement pour cet événement.
  • Langues, outils, systèmes, API et données qui ont été utilisés de manière significative dans le projet.
  • Probabilité de plagiat flagrant.
  • Qualité du code et de la documentation technique.

Étape 3: Au moins trois juges visionneront la vidéo de soumission, examineront les fichiers du projet et liront le rapport de l'étape 2. Chaque juge attribue au projet une note en utilisant la grille de notation qui a été partagée lors de l'annonce du défi.

Étape 4: Les notes des rubriques ne sont pas définitives. Une majorité de juges s'accordent sur les gagnants en utilisant les notes des rubriques comme guide.

Qui peut y assister?

N'importe qui dans le monde peut participer et soumettre des projets! Cependant, il y a des conditions pour les prix en espèces.

Qui peut gagner les prix en espèces?

Vous devez être:

  • Un citoyen canadien ou un résident permanent.
  • Pas un fonctionnaire.

Les non-Canadiens peuvent-ils gagner des prix en argent?

Non. Cependant, les groupes qui mélangent des Canadiens et des non-Canadiens peuvent gagner des prix en argent, sauf que le groupe doit être au moins à moitié canadien et que l'argent du prix ne sera envoyé qu'à eux.

Les fonctionnaires peuvent-ils gagner des prix en espèces?

Non.

Mon projet est-il suffisamment code source ouvert pour le prix?

Pour ce hackathon, votre projet est considéré comme code source ouvert si:

  • Vous avez choisi une licence pour votre projet.
  • Un système comme Linux/Ubuntu peut exécuter votre logiciel sans installer ni payer pour des projets de source fermée supplémentaires.
  • Tout votre code source est accessible au public sur un site web comme GitLab ou GitHub sans nécessiter de connexion.

Si ce n'est toujours pas clair, demandez à un organisateur d'événement.

Cet événement est-il gratuit?

Oui. Cependant, vous devrez vous inscrire (gratuitement) pour soumettre un projet et gagner un prix.
